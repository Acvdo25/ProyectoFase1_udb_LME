# Tarea Desafío 1 UDB-HTML

## Criterios a evaluar

> 
>* Diseño responsive con Bootstrap.

Mediante el desarrollo de este proyecto de cátedra se pretende que el alumno demuestre los conocimientos aprendidos durante el curso y ponga en práctica todas las destrezas adquiridas en cuanto al diseño de sitios web. Los temas de proyecto disponibles para la selección al azar son los siguientes: 

● Diseño de un sitio web para un equipo deportivo. 
● Diseño de un sitio web para una banda o grupo musical. 
● Diseño de un sitio web para una agencia de turismo. 
● Diseño de un sitio web para un cine. 
● Diseño de un sitio web para un restaurante. Para la primera fase del proyecto se solicita lo siguiente: 
● Creación de todas las páginas web mencionadas en las condiciones específicas del proyecto seleccionado (ver al final de esta ruta los temas disponibles) con su respectiva maquetación web en html y su aspecto visual aplicando hojas de estilo css. 

● Utilización de una plantilla creada por los estudiantes que se aplique en todas las páginas que conforman el sitio web, respetando la estructura básica de una página conformada por su cabecera (puede incluir un banner o slider, el logo de la entidad y el menú de navegación), cuerpo (todos los elementos de la página) y pie de página (incluir las redes sociales y alguna información breve de la entidad como sus datos de contacto).

## Levantar proyecto
Tener instalado previamente Git y cuenta creada en Github luego clonar el repositorio utilizando el siguiente comando:
```
git clone https://github.com/DavidGalileo24/desafio1_udb.git
```
Posteri corriendo el siguientormente abrir una consola dentro de la carpeta del proyecto e instalar las dependencias del proyecto donde traera la instalación de Bootstrap y otras que se lleguen a utlizare comando:
```
npm install
```


Instalar la extensión de `Live Server (Five)` en el VSCode y en el index.html que esta afuera en raíz dar clic derecho abrir con Live server para que pueda mostrarse el proyecto.



Se detallan varios comando de git para poder trabajar en su respectiva rama y se hagan los cambios:
```
git checkout <nombre_rama>  (Cambiarse a su rama correspondiente)
git add .  (Agregar los cambios realizados)
git commit -m "Agregar mensaje" (Comando para detallar mensaje de los cambios realizados)
git push  (Subir los cambios a rama remota)
```
